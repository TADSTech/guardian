# Non-Code Deliverables: Design & Brand Assets

This document details the branding, layout, and visual marketing assets that need to be created before launching Guardian.

---

## 🎨 1. Brand Identity & Logo Design
* **Objective**: Establish a trusted, recognizable, and warm brand system that resonates with family caregivers.
* **Deliverables**:
  * [ ] **Logo Mark**: A clean emblem combining a shield (security) and a heart or handshake (empathy/care).
  * [ ] **Color Palette Specs**: Document HSL codes for the Deep Forest Green (`#032f22`), Warm Cream (`#fbfaf5`), Alert Orange, and Soft Sage accents.
  * [ ] **Typography Guide**: Standardize fonts for header styling (e.g., *Outfit* or *Playfair Display*) and body accessibility (e.g., *Inter* or *Open Sans*).

---

## 📱 2. High-Fidelity UI/UX Figma Prototypes
* **Objective**: Map out the complete end-to-end user journeys before coding future screens.
* **Deliverables**:
  * [ ] **Onboarding Screens**: Designing the sponsor signup, payment integration, and dependency pairing wizard in Figma.
  * [ ] **Mobile Responsive Layouts**: Visualizing how the admin dashboard scales down to 320px mobile viewports.
  * [ ] **WhatsApp Bot Templates**: Designing standard structured reply frames to ensure layout consistency.

---

## 📢 3. Marketing & Pitch-off Visuals
* **Objective**: Create visual assets for marketing campaigns and pitch presentations.
* **Deliverables**:
  * [ ] **Pitch Deck Slides**: Design a visually stunning, warm-themed 10-slide deck matching the cream-and-green aesthetic.
  * [ ] **Explainer Graphics**: High-quality diagrams showing the WhatsApp Bot forwarding process and Chrome Extension companion panels.
  * [ ] **Social Media Assets**: Standard post templates demonstrating example scam messages before and after Guardian's analysis.
